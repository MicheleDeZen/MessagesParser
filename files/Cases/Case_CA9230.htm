KSS03199
SETINFO su {Nome variabile KRL} ({Luogo di dichiarazione}) sospeso
Causa:La variabile non esiste o il luogo di dichiarazione non è valido
Descrizione
Impossibile attivare una SETINFO sospesa su una variabile KRL. Con SETINFO le variabili sono lette in dipendenza dagli eventi. I moduli software non sono pertanto più informati su modifiche del valore della variabile.
Per verificare quale variabili abbia causato l’errore:
Istruzione verifica
  1. Aprire il file menzionato nei parametri di notifica Luogo di dichiarazione.
  2. Verificare se la variabile indicata nei parametri di notifica Nome variabile KRL, è dichiarata nel file. Possibili fonti di errori:
    • La dichiarazione è stata cancellata.
    • La riga con la dichiarazione è stata commentata con ;.
    • Il file in cui deve essere presente la dichiarazione non esiste con questo nome oppure il luogo di dichiarazione non è valido.
Rimedio:Annullare le modifiche alle impostazioni standard KUKA
Descrizione
I parametri standard KUKA assegnati, le impostazioni e le parti di programma non possono essere modificati né cancellati. Adattamenti specifici dell’utente possono essere eseguiti solo nelle aree USER apposite.
Esempio per un’area USER prevista appositamente per adattamenti specifici dell’utente:
...
;FOLD INI
  ;FOLD BASISTECH INI
    GLOBAL INTERRUPT DECL 3 WHEN $STOPMESS==TRUE DO IR_STOPM ( )
    INTERRUPT ON 3 
    BAS (#INITMOV,0 )
  ;ENDFOLD (BASISTECH INI)
  ;FOLD USER INI
    ;Make your modifications here
 
  ;ENDFOLD (USER INI)
;ENDFOLD (INI)
...
Procedimento
  • Le modifiche apportate sono annullate e gli adattamenti specifici dell’utente sono eseguiti nelle aree USER.
