KSS10021
{Nome istanza} in attesa di avvio 'esterno' dell'Interbus
Causa:Avvio esterno attivato
Descrizione
In IBSPCI.XML è stato attivato lo start esterno. Di conseguenza non avvengono scambi di dati I/O lunghi tramite Interbus, fino a che non ha luogo un avvio esterno.
Rimedio:Disattivare l’avvio esterno in IBSPCI.XML
Descrizione
Per disattivare l’avvio esterno, è necessario mettere la voce EXTERN_START in ISBSPCI.XML su Falso.
Nota
Valore
Funzione
EXTERN_START
False
True
Default: False
False: L’Interbus è configurato e avviato dal driver.
True: L’Interbus è configurato e avviato da un tool esterno come CMD o da un progetto boot. La voce CMD_FILE è ignorata.
Presupposto
  • Livello Windows (CTRL+ESC)
Procedimento
  1. Aprire il file C:\KRC\ROBOTER\Config\User\Common\IBSPCI.XML.
  2. Controllare le impostazioni applicate nei singoli segmenti e se necessario adattarle.
  3. Salvare le modifiche e chiudere il file.
Le modifiche devono sempre essere acquisite nel menu principale Configurazione > Inputs/outputs > Driver I/O.
