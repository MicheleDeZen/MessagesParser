KSS01422
{Variabile $} valore non valido
Causa:EKI: Variabile non inizializzata
Descrizione
KUKA.Ethernet KRL mette a disposizione funzioni per lo scambio di dati tra controllo robot e sistemi esterni. I dati ricevuti vengono esportati tramite funzioni di accesso dalla memoria EKI interna e scritti su variabili KRL. Le variabili devono essere inizializzate prima dell’uso nella funzione di accesso. Una o più variabili non sono state inizializzate.
Esempio di errata programmazione:
 
DEF Beispielprogramm_Fehler( )
...
REAL valueReal
...
RET=EKI_GetReal("XmlTransmit","Sensor/Positions/Current/@X",valueReal)
END
Esempio di corretta programmazione:
 
DEF Beispielprogramm_Fehler( )
...
REAL valueReal
valueReal=0.0
...
RET=EKI_GetReal("XmlTransmit","Sensor/Positions/Current/@X",valueReal)
END
Per verificare se la variabile è stata inizializzata in modo corretto:
Istruzione verifica
  1. Aprire il programma.
  2. Cercare EKI_Get.
  3. Verificare se la variabile è stata inizializzata prima della funzione di accesso.
Rimedio:Inizializzare la variabile
Descrizione
Inizializzare la variabile prima della funzione di accesso.
Procedimento
  1. Selezionare il file nel navigatore e premere Apri. Il file viene visualizzato nell’editor
  2. Cercare il punto corrispondente nel file e inizializzare la variabile.
  3. Salvare il file e chiuderlo.
    La variabile è inizializzata.
