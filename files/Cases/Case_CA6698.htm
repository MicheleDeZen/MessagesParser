KSS10024
{Nome istanza} Slave-ID della scheda ({Slave-ID della scheda}) diverso da quello nel file XML ({Slave-ID nel file XML})
Causa:ID Slave non corretto
Descrizione
L’ID della scheda Slave non coincide con l’ID Slave indicato in IBSPCI.XML.
Rimedio:Adattare lo slave ID in IBSPCI.XML
Descrizione
L’ID della scheda Slave deve essere assegnato in IBSPCI.XML alla voce SLAVE_ID. L’ID deve coincidere con l’ID della scheda Slave.
Nota
Valore
Funzione
SLAVE_ID
Default: 0x0403
L’ID Slave si compone di due parti: la lunghezza dei dati di processo Slave in High-Byte e l’ID Slave in Low-Byte. Di conseguenza l’ID Slave 3 rappresenta un modulo d’ingresso e uscita digitale.
Se non si specifica alcun ID Slave, si inizializza lo Slave con ID 0x0403. Questa voce non è valutata all’avvio dell’accoppiatore di sistema tramite file SVC o esternamente.
Se si modifica l’ID Slave, il gruppo Slave deve essere brevemente staccato dall’alimentazione esterna da 24V, per poter acquisire i nuovi dati.
Presupposto
  • Livello Windows (CTRL+ESC)
Procedimento
  1. Aprire il file C:\KRC\ROBOTER\Config\User\Common\IBSPCI.XML.
  2. Controllare le impostazioni applicate nei singoli segmenti e se necessario adattarle.
  3. Salvare le modifiche e chiudere il file.
Le modifiche devono sempre essere acquisite nel menu principale Configurazione > Inputs/outputs > Driver I/O.
