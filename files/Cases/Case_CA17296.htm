KSS03329
Assegnazione a $MOTION_MODE non permessa, Motion Mode {Motion Mode specifico} non installata {ModuleLine}
Causa:Modo di simulazione delle KUKA Motion Modes su OfficeLite / OfficePC non configurato
Descrizione
Un modo di simulazione delle KUKA Motion Mode è gratuitamente disponibile in OfficeLite e OfficePC, senza utilizzare pacchetti opzioni installati. Di default le variabili necessarie non sono però configurate opportunamente nel software di sistema, per questo motivo si genera un errore quando la variabile di sistema $MOTION_MODE viene impostata dall’utente su un valore diverso da #STANDARD.
Rimedio:Configurazione del modo di simulazione delle KUKA Motion Mode su OfficeLite / OfficePC
Descrizione
Un modo di simulazione delle KUKA Motion Mode è gratuitamente disponibile in OfficeLite e OfficePC, senza utilizzare pacchetti opzioni installati. Di default le variabili necessarie non sono però configurate opportunamente nel software di sistema.
La simulazione delle Motion Mode viene configurata attraverso una variabile di sistema:
Possibilità di scrittura
La variabile può essere modificata in KRC:\STEU\Mada in $option.dat.
Non può essere modificata dalla visualizzazione delle variabili né dal programma.
Sintassi
$INSTALLED_MOTION_MODES = {PATH state, DYNAMIC state}
Spiegazione della sintassi
Elemento
Descrizione
state
Tipo: BOOL
  • TRUE: la Motion Mode è installata.
  • FALSE: la Motion Mode non è installata.
Default: FALSE
Altre indicazioni:
  • se è stata eseguita una precedente installazione dei pacchetti opzioni Motion Mode, non è più necessario modificare manualmente la variabile di sistema $INSTALLED_MOTION_MODES anche in OfficeLite e OfficePC.
  • La conversione manuale è possibile esclusivamente per il modo di simulazione su OfficeLite e OfficePC.
  • La conversione manuale su un controllo reale produce invece degli errori.
  • In un controllo reale occorre necessariamente eseguire l’installazione dei rispettivi pacchetti opzioni Motion Mode.
