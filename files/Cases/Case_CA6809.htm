KSS01421
{Nome oggetto} : {Numero errore} Errore analisi
Causa:Errore syntax nel file
Descrizione
Nel file vi sono parti non comprese dal comando.
Esempi per file in cui possono verificarsi errori di sintassi:
  • File SRC (programma robot)
  • File DAT (elenco di dati)
  • File SUB (esecuzione nell’interprete submit)
  • File INI (file di configurazione)
Esempi di errori di sintassi sono:
  • Prima di un commento è stato dimenticato il punto e virgola.
  • Errore di scrittura di un comando.
  • Gli argomenti di un comando sono stati inseriti nella sequenza sbagliata.
  • Comando incompleto, ad es. argomenti dimenticati.
  • Sono state utilizzate variabili sconosciute, ad es. un nome di variabile è stato scritto male.
  • La variabile utilizzata si distingue per il tipo di dati dalla variabile prevista.
  • Sono stati utilizzati variabili, funzioni o sottoprogrammi che non vi sono più. Ad es. un pacchetto tecnologico utilizzato in precedenza è stato nel frattempo disinstallato.
Rimedio:Visualizzazione e rimuovi errore di sintassi
Presupposto
  • Gruppo utenti Esperto
  • Non è selezionato alcun programma.
Procedimento
  1. Selezionare il programma errato nel navigatore.
    Se il file errato è sconosciuto, esso sarà legato attraverso un richiamo di sottoprogramma e dovrà prima essere rilevato.
    Per la ricerca sono interessanti tutte le directory sotto KRC:\.
  2. Premere il pulsante Lista errori. Si visualizza l’elenco degli errori.
  3. Selezionare gli errori nell’elenco degli errori.
    Si consiglia di elaborare l’elenco degli errori dall’alto verso il basso. L’errore inferiore potrebbe essere un errore consequenziale.
  4. Premere il pulsante Visualizza. Il programma errato si apre e il cursore indica il punto con errore.
    Se l’errore è nascosto in un FOLD, il cursore è settato sul FOLD non aperto.
    Se il FOLD viene aperto, è possibile ripetere i passaggi 3 e 4 per passare direttamente al punto errato.
  5. Correggere la sintassi KRL errata.
  6. Premere il pulsante Chiudi. Alla richiesta di conferma, se si intendono salvare le modifiche, rispondere con il tasto Sì.
  7. Ripetere la procedura fino a che tutti gli errori di sintassi KRL non sono stati corretti.
