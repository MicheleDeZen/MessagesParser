Simulate 00003
In attesa di {Espressione}
Causa:KRC_Interrupt è settato
Descrizione
Il robot è controllato tramite l’interfaccia KUKA.PLC mxAutomation. Sebbene non vi siano messaggi di errore non può essere movimentato molto. È stato settato un KRC_Interrupt.
Per controllare se KRC_Interupt è settato:
Istruzione verifica
  • Controllare se è presente il seguente messaggio con parametri di notifica:
    Attendere {(( non iMxA_BRAKE) e ( non iMxA_BRAKE_F ) e iMxA_RELEASE_BRAKE)}
Moduli funzionali:
Descrizione
Con il modulo funzionale KRC_Interrupt si arresta il robot. Esso frena delicatamente (BRAKE) o nel modo più rapido possibile (BRAKE F) da velocità elevate.
Se si attiva un’istruzione BRAKE, non si elaborano più istruzioni tramite l’interfaccia mxA.
Anche il modulo funzionale KRC_Abort non viene più elaborato. KRC_Abort può interrompere il programma solo se è stato continuato con KRC_Continue e pertanto l’istruzione BRAKE non è più attiva.
Mentre l’istruzione BRAKE è attiva, il programma può essere interrotto solo tramite RESET sul modulo funzionale KRC_AutomaticExternal.
Modulo funzionale KRC_Interrupt
Descrizione
Con il modulo funzionale KRC_Continue è possibile continuare un programma interrotto tramite Interrupt.
Se si utilizza KRC_Continue insieme a KRC_Interrupt, l’ingresso Execute con KRC_Interrupt deve avere il valore FALSE, prima di poter eseguire KRC_Continue.
Modulo funzionale KRC_Continue
Rimedio:Richiamo di KRC_Continue
Descrizione
Con il modulo funzionale KRC_Continue è possibile continuare un programma interrotto tramite Interrupt.
Se si utilizza KRC_Continue insieme a KRC_Interrupt, l’ingresso Execute con KRC_Interrupt deve avere il valore FALSE, prima di poter eseguire KRC_Continue.
Per guidare di nuovo il robot tramite KUKA.PLC mxAutomation, è necessario richiamare il modulo PLC KRC_Continue.
