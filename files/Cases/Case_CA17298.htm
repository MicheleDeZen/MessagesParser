KSS03329
Assegnazione a $MOTION_MODE non permessa, Motion Mode {Motion Mode specifico} non installata {ModuleLine}
Causa:A $MOTION_MODE è stata assegnata una modalità non installata.
Descrizione
A $MOTION_MODE è possibile assegnare un valore solo se è installata la corrispondente KUKA Motion Mode.
Esempio:
L’assegnazione $motion_mode = #path è possibile solo se è installato il pacchetto opzioni KUKA.PathMode.
Rimedio:Installare la KUKA Motion Mode da WorkVisual.
Descrizione
Deve essere installata una KUKA Motion Mode in WorkVisual e deve essere aggiunta al progetto. Durante il trasferimento del progetto al controllo robot, il pacchetto opzioni viene automaticamente installato sul controllo robot.
Con l'update è prima necessario disinstallare la precedente versione del pacchetto opzioni in WorkVisual.
Prima di procedere all'aggiornamento di un software, si consiglia di archiviare tutti i dati pertinenti.
Presupposto
  • KSS: Gruppo utenti Esperto
    VSS: Gruppo utenti Utente
  • Modo operativo T1 o T2
  • Non è selezionato nessun programma.
  • Connessione di rete al controllo robot
  • Il pacchetto opzioni si presenta come file KOP.
Procedimento
  1. Solo in caso di aggiornamento: Disinstallare la precedente versione del pacchetto opzioni Motion Mode … in WorkVisual.
  2. Installare il pacchetto opzioni Motion Mode … in WorkVisual.
  3. Caricare il progetto attivo dal controllo robot.
  4. Aggiungere al progetto il pacchetto opzioni Motion Mode ….
  5. Configurare il pacchetto opzioni in WorkVisual come necessario.
  6. Trasferire il progetto da WorkVisual sul sistema di controllo robot e attivarlo.
  7. Sulla smartHMI viene visualizzata la richiesta di conferma Si vuole autorizzare l'attivazione del progetto […]?. Al momento dell'attivazione, il progetto attivo verrà sovrascritto. Se non è stato sovrascritto alcun progetto rilevante: Confermare la richiesta con Sì.
  8. Sullo smartHMI viene visualizzata una panoramica con le modifiche e una richiesta di conferma. Rispondere con Sì. Il pacchetto opzioni viene installato e il controllo robot esegue un riavvio.
Per informazioni relative ai cicli in WorkVisual, consultare la documentazione di WorkVisual.
File LOG
Un file LOG viene creato in C:\KRC\ROBOTER\LOG.
