KSS13038
Impossibile avviare stack del dispositivo Profinet, codice errore: {Codice}
Causa:Il file non è corretto
Descrizione
Il file pndev1.xml non è corretto.
Rimedio:Installare PROFINET (tramite WorkVisual)
Descrizione
Il pacchetto opzioni viene installato in WorkVisual e inserito nel progetto. Durante il trasferimento del progetto al controllo robot, il pacchetto opzioni viene automaticamente installato sul controllo robot.
Con l'update è prima necessario disinstallare la precedente versione del pacchetto opzioni in WorkVisual.
Prima di procedere all'aggiornamento di un software, si consiglia di archiviare tutti i dati pertinenti.
Presupposto
  • Gruppo d'utente Esperto
  • Modo operativo T1 o T2
  • Non è selezionato nessun programma.
  • Connessione di rete al controllo robot
  • Il pacchetto opzioni si presenta come file KOP.
Procedimento
  1. Solo in caso di aggiornamento: Disinstallare la precedente versione del pacchetto opzioni KUKA.PROFINET MS in WorkVisual.
  2. Installare il pacchetto opzioni KUKA.PROFINET MS in WorkVisual.
  3. Caricare il progetto dal sistema di controllo robot.
  4. Aggiungere al progetto il pacchetto opzioni KUKA.PROFINET MS.
  5. Configurare il pacchetto opzioni.
  6. Trasferire il progetto da WorkVisual sul sistema di controllo robot e attivarlo.
  7. Viene visualizzata la richiesta di conferma Si vuole autorizzare l'attivazione del progetto [...]?. Al momento dell'attivazione, il progetto attivo verrà sovrascritto. Se non è stato sovrascritto alcun progetto rilevante: Confermare la richiesta con Sì.
  8. Sulla smartHMI vengono visualizzati una panoramica con le modifiche e una richiesta di conferma. Rispondere con Sì. Il pacchetto opzioni viene installato e il controllo robot esegue un riavvio.
Per informazioni relative ai cicli in WorkVisual, consultare la documentazione di WorkVisual.
File LOG
Un file LOG viene creato in C:\KRC\ROBOTER\LOG.
