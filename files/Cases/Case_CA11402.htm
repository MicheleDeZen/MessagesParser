KSS01409
Nessun Trigger permesso in ISR oppure *.SUB
Causa:Istruzione TRIGGER non ammessa all'interno di un programma SUB.
Descrizione
Un programma SUB può svolgere attività di comando e monitoraggio. Esempi: Monitoraggio di dispositivi di protezione; monitoraggio di un circuito di raffreddamento.
In un simile programma SUB (ad es. SPS.SUB) è stata utilizzata un’istruzione TRIGGER inammissibile.
Codice KRL
In un sottoprogramma è possibile utilizzare quasi tutte le istruzioni KRL. Non sono tuttavia possibili le seguenti istruzioni:
  • Istruzioni per i movimenti del robot
    I movimenti del robot possono essere interpretati solo dal Roboter-Interpreter. Per questo motivo da un sottoprogramma non possono essere richiamati programmi SRC contenenti istruzioni di movimento come sottoprogrammi.
  • Istruzioni che si riferiscono a movimenti del robot
    Vi rientrano BRAKE e tutti i TRIGGER.
Le istruzioni di movimento per gli assi aggiuntivi possono essere utilizzate invece in un sottoprogramma. Esempio:
IF (($IN[12] == TRUE) AND ( NOT $IN[13] == TRUE)) THEN
$VEL_EXTAX[2]=10
$ACC_EXTAX[2]=10
ASYPTP {E2 45}
...
IF ((NOT $IN[12] == TRUE) AND  ($IN[13] == TRUE)) THEN
$VEL_EXTAX[2]=10
$ACC_EXTAX[2]=10
ASYPTP {E2 0}
L’asse aggiuntivo E2 viene movimentato in funzione di determinati ingressi.
Le istruzioni WAIT o le code non sono state qui utilizzate, perché arrestano il ciclo.
Rimedio:Modificare il programma.
Presupposto
  • Gruppo utenti Esperto
  • Modo operativo T1, T2 o AUT
Procedimento
  1. Selezionare il programma nel navigatore e premere Apri. Il programma viene visualizzato nell'editor.
  2. Cercare ed elaborare il punto corrispondente nel programma.
  3. Chiudere il file e alla richiesta di conferma, se si intende salvare le modifiche, rispondere con Sì.
