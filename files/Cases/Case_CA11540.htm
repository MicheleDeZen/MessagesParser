KSS00082
Arresto del robot tramite Submit
Causa:Funzione ROB_STOP() richiamata nel programma Submit
Descrizione
In un programma Submit (ad es. sps.sub) è stata richiamata la funzione ROB_STOP().
  • ROB_STOP() arresta il robot e impedisce altri movimenti. Ciò riguarda tutti i movimenti possibili, che siano dovuti a cicli di programmazione, procedimenti manuali o modalità comando.
  • ROB_STOP_RELEASE() rimuove un blocco causato da ROB_STOP().
Se ROB_STOP() viene richiamato più volte di seguito senza che nel frattempo si richiami ROB_STOP_RELEASE(), solo il primo tentativo è efficace. Gli altri non hanno alcun effetto.
Per verificare se è scattato lo stop a causa di ROB_STOP():
Presupposto
  • Gruppo d'utente Esperto
File di configurazione
Directory
C:\KRC\Roboter\KRC\R1\System
File
sps.sub
Istruzione verifica
  1. Avviare l’interprete submit nel navigatore.
  2. Controllare se al suo interno la funzione ROB_STOP() viene richiamata senza che dopo si richiami ROB_STOP_RELEASE().
  3. Se si utilizzano altri programmi Submit, controllare anche essi.
Rimedio:Richiamare ROB_STOP_RELEASE() nel programma Submit
Descrizione
Correggere il programma Submit in questione.
Presupposto
  • Il navigatore è visualizzato.
  • Gruppo d'utente Esperto
  • Il submit interpreter è deselezionato.
Procedimento
  • Qualora non disponibile nel programma, programmare ROB_STOP_RELEASE() dopo ROB_STOP().
  • Se ROB_STOP_RELEASE() è già disponibile nel programma, rimuovere la causa per la quale ROB_STOP_RELEASE() non è stato richiamato.
