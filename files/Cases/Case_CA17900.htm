KSS26091
Superato limite max. variabile ritardo di posizionamento ({Azionamento}).
Causa:Parametro PosLagSoft configurato male nei dati della macchina per la pinza
Descrizione
Il parametro PosLagSoft, che determina il valore limite del ritardo di posizionamento morbido è configurato male per l’applicazione.
In caso di Override basso nel modo operativo T1 e T2 (ca. 10 %) la posizione pianificata e quella reale si scostano temporalmente e l’errore rimorchio di posizionamento cresce in modo continuo.
Il monitoraggio scatta dopo l’accensione della regolazione della forza sulla soglia di attivazione della forza.
Il limite di monitoraggio è regolato in un file di configurazione:
File di configurazione
Directory
C:\Config\User\Common\Mada\NGAxis
File
Cg1.xml
Sintassi
PosLagSoft="1440"
Il parametro ha l’unità gradi. Il valore predefinito di 1440 corrisponde pertanto a 4 giri del motore.
Per verificare quale limite di monitoraggio è configurato:
Istruzione verifica
  1. Selezionare il file nel navigatore e premere Apri. Il file viene visualizzato nell'editor.
  2. Cercare la variabile o il parametro e verificare il valore.
Rimedio:Aumentare il limite di monitoraggio nel file di configurazione
Descrizione
Aumentare il valore massimo consentito per l'errore di posizionamento morbido di un fattore 10, quindi a 14400. Questo è il valore che verrà utilizzato di default in una nuova versione del software.
File di configurazione
Directory
C:\Config\User\Common\Mada\NGAxis
File
Cg1.xml
Sintassi
PosLagSoft="1440"
Il parametro ha l’unità gradi. Il valore predefinito di 1440 corrisponde pertanto a 4 giri del motore.
Procedimento
  1. Selezionare il file nel navigatore e premere Apri. Il file viene visualizzato nell'editor.
  2. Cercare il punto corrispondente nel file e modificare il valore.
  3. Salvare il file e chiuderlo.
