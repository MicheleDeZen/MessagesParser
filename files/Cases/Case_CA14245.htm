Errore archiviazione
Causa:Presenti file *.dmp di dimensioni eccessive
Descrizione
In caso di errore nel processo di boot tra Windows e VxWorks, viene creato il file cross.dmp e/o kuka_hmi_exe.dmp. Poiché i file e si trovano nella directory log, vengono memorizzati anch’essi al momento della creazione di un KRCDiag e/o di un archivio.
Se l’archivio deve essere creato per un ulteriore diagnosi degli errori, sono assolutamente necessari i file cross.dmp e/o kuka_hmi_exe.dmp.
File
Directory
C:\KRC\Roboter\LOG
File
cross.dmp
kuka_hmi_exe.dmp
Per verificare se il file è stato creato:
Istruzione verifica
  1. Aprire Windows Explorer.
  2. Navigare nella directory nella quale si dovrebbe trovare il file.
  3. Verificare se il file è presente nella directory.
Rimedio:Update del software KSS
Descrizione
Prima di procedere all'aggiornamento di un software, si consiglia di archiviare tutti i dati pertinenti. Se necessario è possibile ripristinare la vecchia versione. Inoltre si consiglia di archiviare lo stato attuale dopo l'update.
Questa funzione non può essere utilizzata per installare nuove versioni, ad es. da KSS 8.1 a KSS 8.2.
Questa funzione non può essere utilizzata per installare update di software aggiuntivi, ad es. di pacchetti tecnologici.
Quando dopo un update si ricaricano i dati macchina, lo stato dei dati macchina deve corrispondere esattamente allo stato del KSS. Ciò è garantito quando i dati macchina dell'update KSS vengono utilizzati.
Se vengono caricati i dati macchina sbagliati, il robot non deve essere messo in movimento! Sussiste altrimenti il rischio di lesioni fisiche e danni materiali.
È possibile utilizzare esclusivamente la chiavetta KUKA.USBData. Impiegando una chiavetta USB diversa, si corre il rischio di perdita o modifica dei dati.
Requisito
  • Software su chiavetta KUKA.USBData
    Il software deve essere copiato sulla chiavetta, in modo che il file Setup.exe sia al livello più alto. (Quindi non in una cartella.)
Procedimento
  1. Inserire la chiavetta USB.
  2. Nel menu principale selezionare Messa in servizio > Software Update > Automatico.
  3. Viene visualizzata una domanda di sicurezza se effettivamente debba essere effettuato un update. Confermare con Sì.
    Nella finestra di segnalazione viene visualizzato il seguente messaggio: Aggiornamento software pronto, riavviare il computer!
  4. Nel menu principale selezionare Spegnimento, quindi selezionare l'opzione Riavviare il PC di comando. (Rinnova lettura file non è necessario)
  5. Confermare la richiesta di conferma con Sì. Il controllo robot si riavvia ed esegue l'update.
    Successivamente il controllo robot esegue ancora una volta un riavvio.
  6. Se il controllo robot ha eseguito il secondo riavvio, si può togliere la chiavetta USB.
Il software di sistema aggiornato è ora disponibile.
