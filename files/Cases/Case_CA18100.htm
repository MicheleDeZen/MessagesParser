KSS26252
Inizializzazione dell'azionamento errata ({Numero apparecchio}).
Causa:File EncoderMastering.xml non scritto correttamente allo spegnimento
Descrizione
Impossibile spegnere correttamente il comando robot, ad esempio per batterie difettose.
Per questo motivo non è stato possibile scrivere completamente il file EncoderMastering.xml. Il file è vuoto o non completo. Il comportamento si verifica solo con robot SE (KR 300 R2500 ultra SE).
File di configurazione
Directory
C:\KRC\Roboter\Config\User\Common\PersistPC\
File
EncoderMastering.xml
Per verificare se il file non è completo:
Presupposto
  • Gruppo d'utente Esperto
Istruzione verifica
  1. Aprire Windows Explorer o Navigator.
  2. Navigare nella directory nella quale si trova il file.
  3. Aprire il file.
  4. Confrontare il contenuto del file con il file allegato.
Rimedio:Sostituire il file e calibrare il robot
Descrizione
Sostituire il file con uno definitivo e poi calibrare il robot.
Presupposto
  • Gruppo d'utente Esperto
Procedimento
  1. Copiare il file allegato da una chiavetta USB.
  2. Copiare il file dalla chiavetta USB nella giusta directory sul comando robot.
  3. Eseguire un avvio a freddo con “Ricarica dati”.
  4. Calibrare il robot.
Procedimento alternativo
  1. Prelevare il progetto WorkVisual dal comando.
  2. Copiare il file allegato nel progetto WorkVisual.
  3. Trasferire nuovamente il progetto al comando.
  4. Eseguire un avvio a freddo con “Ricarica dati”.
  5. Calibrare il robot.
Negli impianti esistenti, il tipo di calibrazione da utilizzare (standard o con correzione del carico) deve corrispondere a quello utilizzato alla messa in servizio. L'utilizzo del tipo di calibrazione sbagliato può portare a errori nella calibrazione del robot.
