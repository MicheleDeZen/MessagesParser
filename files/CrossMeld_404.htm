KSS00404
Arresto di sicurezza
Effetto
ARRESTO D'EMERGENZA lungo il percorso
Immissione di comandi attivi (movimenti robot, avvio programma) bloccata.
Causa o cause possibili
1.Causa:Il Safety ID PROFINET di KR C4 non coincide con il Safety ID del PLC (F_Dest_Add)
Rimedio:Adattare il programma PLC
2.Causa:Consenso esterno non cablato
Rimedio:Eliminare l’errore di cablaggio
3.Causa:Impossibile assegnare automaticamente l’indirizzo FSoE
Rimedio:Settare l’indirizzo FSoE manualmente con WorkVisual, fino a KSS 8.2.21
4.Causa:Il Safety ID PROFINET di KR C4 non coincide con il Safety ID del PLC (F_Dest_Add)
Rimedio:Adattamento dell'ID Safety del PLC (F_Dest_Add) nel portale TIA
5.Causa:Il Safety ID PROFINET di KR C4 non coincide con il Safety ID del PLC (F_Dest_Add)
Rimedio:Adattare l’ID Safety PROFINET nella configurazione di sicurezza del KR C4
6.Causa:Impossibile assegnare automaticamente l’indirizzo FSoE
Rimedio:Settare l’indirizzo FSoE manualmente con WorkVisual, da KSS 8.2.21, KSS 8.3
7.Causa:Nessuna interfaccia di sicurezza configurata
Rimedio:Rettifica della configurazione con WorkVisual
8.Causa:L’asse accoppiabile è stato configurato prima dell’asse permanentemente presente
Rimedio:Rettifica della configurazione con WorkVisual
9.Causa:Il finecorsa software può essere impostato più alto dell’angolo massimo dell’asse
Rimedio:Update del software KSS
10.Causa:CIB Safety Modul (SION-CIB) mancante nel progetto WorkVisual
Rimedio:Rettifica della configurazione con WorkVisual
11.Causa:Collegamento errato tra SIB e l'interfaccia di sicurezza
Rimedio:Realizzazione di collegamento corretto
